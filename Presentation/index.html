Guess & Survive

A classic Hangman game built with C++ featuring strategic gameplay, beautiful ASCII art, and interactive mechanics

About The Project

This is a console-based Hangman game developed in C++ for an educational project at the University of Central Punjab. The game challenges players to guess a hidden word letter by letter before running out of incorrect attempts.

The project demonstrates fundamental programming concepts including string manipulation, vector containers, algorithm design, console I/O, and game flow control through an engaging and interactive experience.

Project Details

  • Language: C++ (Windows Console)
  • Type: Educational Game
  • Institution: University of Central Punjab
  • Course: BCCS-B1 | Fall 2025
  • Difficulty Level: Intermediate

Project Overview

Hangman is a classic word-guessing game where players discover a hidden word by guessing letters one at a time. This implementation combines strategic gameplay mechanics with a polished user interface using Windows console features.

Key Objectives

  • Implement dynamic word selection from a predefined list
  • Manage game state with incorrect guess tracking
  • Provide interactive user feedback with color-coded output
  • Implement special features like the "Deal System"
  • Create engaging ASCII art for visual representation

Game Mechanics

Players have 7 attempts to guess the word. With each incorrect guess, a hangman figure is progressively drawn. The special "Deal System" allows players to reveal a letter after 2 consecutive wrong guesses by sacrificing 2 lives, adding strategic depth to the gameplay.

Algorithm Flow

The game follows a structured algorithm to manage game state and player interactions:

Game Loop Process

  1. Initialize: Select a random word and setup game variables
  2. Display State: Show current word progress with underscores for hidden letters
  3. Accept Input: Get single character from player using _getch()
  4. Validate: Check if letter was already guessed
  5. Match Check: Search for letter in secret word and reveal all instances if found
  6. Update State: Increment incorrect counter or reset consecutive wrong counter
  7. Special Deal: After 2 consecutive wrongs, offer to reveal a letter for 2 lives
  8. Check Outcome: Determine if game should continue, win, or lose
  9. Display Result: Show final hangman state and game outcome

Core Components

  • Word Selection: Random index from predefined word vector
  • Guess Validation: Vector search for duplicate letters
  • Display System: String manipulation with color-coded output
  • ASCII Art: Progressive hangman visualization based on wrong guesses
  • Game State Management: Tracking of lives, guesses, and game status

string selectRandomWord() {
    vector<string> wordList;

}
